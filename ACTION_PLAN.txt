WORKFLOW PRINCIPLE
------------------
The test/offline-sync-simulator branch is LOCAL TESTING ONLY. It is a sandbox
for manually verifying offline sync behavior. No changes from this branch are
merged upstream. All implementation, versioning, and production work starts
from and targets the 'develop' branch.

The simulator branch does NOT need to be version-agnostic or SOLID-compliant.
It only needs to function correctly for local manual QA. The only constraint
is: it must not contaminate production code when developers switch branches.
